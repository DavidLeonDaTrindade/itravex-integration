# Documentacion tecnica

## Objetivo
Esta seccion centraliza la documentacion tecnica de `itravex-integration` con foco en arquitectura, modulos funcionales, capa de datos, integraciones y operacion.

## Indice
- [Arquitectura](architecture.md)
- [Modulos funcionales](modules.md)
- [Capa de datos](data-layer.md)
- [Integraciones y procesos batch](integrations-and-jobs.md)
- [Operacion y despliegue](operations.md)
- [Testing y validacion](testing.md)

## Lectura recomendada segun necesidad
- Si quieres entender la app de arriba abajo: empieza por `architecture.md` y luego `modules.md`.
- Si el problema es de una tabla, relacion o conexion: ve a `data-layer.md`.
- Si el problema toca GIATA, SAMO, sesiones Itravex o `artisan`: ve a `integrations-and-jobs.md`.
- Si el problema es de entorno, Docker, logs o produccion: ve a `operations.md`.
- Si necesitas validar cambios o ampliar cobertura: ve a `testing.md`.

## Alcance
La documentacion describe el estado actual del codigo del repositorio. No sustituye al codigo fuente ni a `docs/codex-context.md`, sino que lo complementa con una vista tecnica mas estructurada.
